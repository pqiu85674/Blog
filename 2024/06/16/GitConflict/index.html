Git Conflict

編輯 index.html

推上 Github

建立另外一個資料夾並 clone 下來

更新原本 dir 的 index.html 內容

把 dir 的 index.html push 到 Github

更新後來建立 dir2 的 index.html 內容

把 dir2 的 index.html push 到 Github

這裡出現 conflict 錯誤，因為前面 dir 已經更新 index.html 的內容並且 push 到 Github，導致 dir2 要 push 時衝突到，注意這裡是因為 dir2 更改的內容剛好與 dir 的內容不一致所導致

這邊可以解決 conflict ， VS code 會出現兩者不一致的內容讓我們做選擇，編輯好後就可以 push 到 Github

恭喜成功！！！
