Git Branch

假設 A 電腦跟 B 電腦都共用一個 Repositories
在 A 及 B 電腦上先clone
以下方便區分 A 及 B 電腦
所以將 A 電腦路徑設為./gitA/

B 電腦路徑設為./gitB/

B 電腦 clone 下來後 A 電腦 對檔案內容進行更改
更改完後執行 git add . git commit -m 'A Message' git push
push 完之後看一下 log git log --oneline 可以看到 commit 的紀錄
這邊是故意先 push 讓其他使用者的 git log不同

開發告一段落後 push 上 Github

剛建立好 branch 直接 push 會失敗，因為電腦不知道你要 push 哪個 branch，所以要設定git push -u origin HEAD同等於git push -u origin new-branch-name告訴電腦要 push 的 branch，HEAD 會抓到電腦的 branch 最後一次 commit，也可以直接打 branch 的名稱

Github

到 Github 看你的 branch 就會看到兩個 branch
可以切換到 new-branch-name 點開 commit 可以看到 B 電腦所 push 的內容，也可以查看剛剛 B 電腦修改的檔案

這邊沒辦法直接把新的 branch 合併到原本的 branch

會顯示檔案衝突要手動解決檔案的內容，這邊是使用 VS Code 直接開啟檔案就可以對檔案進行更改
修改完檔案後

這裡git push失敗的原因是因為剛剛我們在 B 電腦上把 A 電腦的修改給 pull 到 B 電腦上進行合併，所以目前的 B 電腦 git log 是 main 的 commit 合併 new-branch-name 跟 Github 裡面 new-branch-name 的 commit 不一樣，所以不能git push

這裡要用 git push -f 強制 push，就可以把我們合併的 commit 給 push 到 Github 的 new-branch-name

注意

這裡使用 git push -f 是將合併好的 commit 給 push 到 B 電腦所創造出來的 new-branch-name分支上
但是git push -f 千萬不要在主要分支做使用，只能在自己建立的 branch 分支上做使用，因為此指令會更改歷史紀錄，一旦在主要分支上使用會更改到別人的紀錄，會非常混亂

再次回到 Github 並切換到 new-branch-name 點開 commit 可以看到已經成功更新了
看好之後切回 new-branch-name 可以看到上面綠色的Compare & pull request點擊下去
點擊 Create pull request
可以看到上面有base可以選擇main(原本的 branch) compate 選擇 new-branch-name(新的 branch)
看到以下訊息
Able to merge. These branches can be automatically merged.
代表已經可以把新建立的分支放到主要分支上了
按下 Create pull request
按下 Merge pull request
按下 Confirm merge
這樣就成功了
可以查看 main 裡面的 commit

統整

A 電腦先 push 上 Github,B 電腦 clone 下來，A 電腦改變內容再次 push 上 Github
B 電腦創造新的 branch，在新的 branch 上開發，B 開發完成之後，B 電腦回到自己原本的 Branch(main)上
輸入指令：git pull 把最新的紀錄抓到 B 電腦的原本的 Branch 上
註：這裡有可能出現 Conflict，因為這裡 A 電腦已經更改內容並且 push 上 Github，而 B 電腦也有可能更改 Branch(main)並且 commit
假如出現 Conflict 要輸入指令：

回到 B new-branch-name

到 Github 產生 pull repuest 並且 Merge
