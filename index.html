constructor

ES5

  • constructor(建構函數)通常會以大寫字母開頭，這是一種命名慣例，以便與普通函數區分開來
  • this 會指向新創建的物件 你可以在建構函數內部使用 this 來設置物件的屬性
  • 使用 new 關鍵字以創建新的物件實例

要注意的點

  • 不應該把 function 直接寫在函式建構式當中，否則每個被 new 出來的物件都包含此 function，這樣非常浪費記憶體空間
  • Person.prototype 上定義的方法可以被所有 new 出來的物件所共享，這樣可以節省記憶體空間

繼承

ES5

講解

  • 先宣告 Dog function 使用 const _變數名稱 並使用 this.function 來取得內部資料 來實現封裝

  • Dog.prototype.run把 function 建立在 prototype 上，使其不會跟隨 new 出來的物件
    來達到記憶體優化效果

  • 再宣告 DogChild function 這裡參數要放入想父層的參數，也就是 name 和 color，加上本身的參數 age

  • Dog.call(this,name,color);這裡是放父層的資訊

  • DogChild.prototype = Object.create(Dog.prototype);
    這行是讓 DogChild 繼承 Dog 的屬性及方法
    其中Object.create(Dog.prototype);會建立一個新的對象
    這個對象的原形是 Dog.prototype，所以此對象會繼承 Dog 的方法及屬性
    而DogChild.prototype被賦予新的對象，這樣 DogChild 的所有實例就能夠訪問 Dog 原型上的方法

  • DogChild.prototype.constructor = DogChild;
    使用DogChild.prototype = Object.create(Dog.prototype);的時候
    DogChild.prototype.constructor被設為 Dog
    因為 DogChild.prototype 的原型已經被改變了
    這代表 DogChild 實例的 constructor 屬性會指向 Dog 而不是 DogChild

    這行程式的作用是手動修正 DogChild.prototype.constructor 為 DogChild
    這樣 DogChild 的實例就可以引用它們的構造函數 DogChild

  • 後續宣告const dogChild = new DogChild("dog", "black", 18);
    dogChild 就可繼承 Dog 方法及物件

並且在 constructor 裡面使用 super 來引入上層 class 參數

    程式碼說明

    http： 引入 Node.js 的 HTTP，用於創建伺服器
    headers： 定義 CORS，允許跨域請求

    “Access-Control-Allow-Headers”
    作用：指定哪些 HTTP 的 Headers 設定可以在跨網域請求中發送到我的伺服器
    内容：”Content-Type, Authorization, Content-Length, X-Requested-With”
    這意味著客戶端可以在後續的實際請求中使用 Content-Type, Authorization, Content-Length, X-Requested-With

    “Access-Control-Allow-Origin”
    作用：指定允許哪些網域來源可以造訪我的伺服器
    内容：*
    表示所有的網域都可造訪我的伺服器

    “Access-Control-Allow-Methods”
    作用：指定允許客戶端使用的 HTTP 方法
    内容：”PATCH, POST, GET,OPTIONS,DELETE”
    表示允許使用 PATCH、POST、GET、OPTIONS 和 DELETE 方法。

    “Content-Type”
    作用：指定回傳內容型別
    内容：”application/json”
    客戶端回傳的數據型別為 JSON 格式

    requestListener ： HTTP 的 Request 以及 Response，狀態碼設為 200，內容為 Hello
    http.createServer(requestListener) ： 創建伺服器
    port ： 從環境變量讀取 port，如果沒有設置則默認使用 8000 port
    server.listen ： 啟動伺服器並且監聽 port 啟動成功輸出 Server is running on port 8000
    shutdown ： 定義一個函數，當接收到退出信號時關閉伺服器並退出
    process.on("SIGTERM", shutdown)和process.on("SIGINT", shutdown) ： 監聽終止信號，並調用 shutdown 關閉伺服器

    在nodemon server.js啟動伺服器想要關閉時按下ctrl+c就是觸發process.on("SIGINT", shutdown)
    而我也可以在終端機輸入kill -9 00000強置關閉你正在使用的 port 會觸發process.on("SIGTERM", shutdown)
    而使用這兩種SIGTERM和SIGINT可以優雅且關閉伺服器且沒有任何數據丟失或損壞非常重要
